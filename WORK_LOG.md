## 1. Website (React SPA)

### Architecture
```
src/
├── App.jsx              # Router: /, /presentations
├── App.css              # All styles
├── pages/
│   └── Presentations.jsx  # Paper presentation signup page
├── data/
│   ├── curriculum.js      # 16-week course schedule
│   ├── readingList.js     # All papers (including foundational 2022-2024)
│   └── presentationPapers.js  # 2025-2026 only papers for presentations
├── components/
│   └── Nav.jsx            # Navigation bar
api/
├── _db.js                 # Neon DB connection (DATABASE_URL env var)
├── setup.js               # DB schema creation endpoint
├── presentations/
│   ├── index.js           # GET (list all) + POST (create) + DELETE (remove)
│   └── [id].js            # (legacy, not used after routing fix)
```

### Key Pages

#### Course Home (`/`)
- 16-week curriculum display with topics, dates, descriptions
- Reading list organized by week/subtopic
- Project ideas section
- Responsive design with Inter/Inter Light fonts

#### Paper Presentations (`/presentations`)
- Students sign up for paper presentation subtopics
- Tab-based UI: W2-W15 (W1/W8/W16 excluded)
- Each subtopic shows 5+ recent (2025-2026) papers
- One student per subtopic, one subtopic per student
- Confirm dialog warns about one-time signup
- Remove assignment (×) button with confirmation

### Database Schema
```sql
CREATE TABLE presentations (
  id SERIAL PRIMARY KEY,
  week TEXT NOT NULL,
  topic_idx INTEGER NOT NULL,
  subtopic_idx INTEGER NOT NULL,
  student_name TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(topic_idx, subtopic_idx)
);
```

### API Endpoints
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/presentations` | List all assignments |
| POST | `/api/presentations` | Create assignment (validates: 1 student = 1 subtopic) |
| DELETE | `/api/presentations?id=X` | Remove assignment |
| GET | `/api/setup` | Create DB tables |

### Data Files

#### `presentationPapers.js` (2025-2026 only)

## 2. Bugs Fixed

### Remove Assignment Bug (Critical)
**Problem**: The × button to remove presentation assignments didn't work (HTTP 405).

**Root Cause**: Vercel routing conflict — `api/presentations.js` (file) shadowed the `api/presentations/` directory, so `DELETE /api/presentations/123` went to `presentations.js` instead of `presentations/[id].js`.

**Fix (two commits)**:
1. Moved `api/presentations.js` → `api/presentations/index.js` (fixed import path `../_db.js`)
2. Moved DELETE handler INTO `index.js` using query param: `DELETE /api/presentations?id=X`
3. Updated frontend `handleRemove` to use `?id=${id}` query param

### Error Messages (i18n)
- Changed Korean error messages to English in API handlers
- "Each student can only present one subtopic. Please remove your current assignment first."
- "This subtopic has already been assigned."

## 4. Deployment

### Vercel Configuration
```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/$1" },
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

### Environment Variables (Vercel)
- `DATABASE_URL`: Neon PostgreSQL connection string

### Deployment Process
1. `npm run build` (Vite)
2. `git push origin main` (triggers Vercel auto-deploy)
3. Or: `npx vercel --prod` (manual deploy)
